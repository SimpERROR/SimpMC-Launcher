# 小组件编写文档

## 概述

小组件是 SimpMC 启动器中的可扩展功能模块，可以在主页上显示各种信息和提供快捷操作。本文档详细介绍如何编写和配置小组件。

## 小组件结构

每个小组件都是一个 JSON 对象，包含以下字段：

```json
{
  "type": "unique_identifier",
  "author": "作者名称",
  "name": "显示名称",
  "size": "尺寸",
  "desc": "描述",
  "category": "分类",
  "version": "版本号",
  "html": "HTML模板",
  "styles": "CSS样式",
  "config": [], // 配置选项
  "permissions": [], // 权限列表
  "render": "渲染函数",
  "scripts": "初始化脚本",
  "cleanup": "清理函数"
}
```

## 字段说明

### 基本信息

- **type**: 唯一标识符，必须唯一，用于区分不同的小组件
- **author**: 作者名称
- **name**: 显示在界面上的名称
- **size**: 小组件尺寸，支持 "1x1", "1x2", "2x1", "2x2"
- **desc**: 小组件功能的简要描述
- **category**: 分类，用于在商店中分组
- **version**: 版本号，格式为 "x.y.z"

### 模板和样式

- **html**: HTML 模板字符串，用于定义小组件的结构
- **styles**: CSS 样式字符串，用于美化小组件外观

### 配置选项

`config` 是一个数组，每个配置项包含：

```json
{
  "key": "配置键名",
  "label": "显示标签",
  "type": "输入类型",
  "default": "默认值",
  "options": [], // 对于 select 类型
  "min": 0, // 对于 number 类型
  "max": 100 // 对于 number 类型
}
```

支持的输入类型：
- `text`: 文本输入
- `number`: 数字输入
- `checkbox`: 复选框
- `color`: 颜色选择器
- `select`: 下拉选择

### 权限系统

`permissions` 数组定义小组件需要的权限：
- `network`: 网络访问权限
- `storage`: 本地存储权限

### 脚本函数

#### render 函数
可选的渲染函数，用于动态生成 HTML：

```javascript
"render": "return `<div>动态内容：${config.someValue}</div>`;"
```

在函数中可以使用：
- `config`: 用户配置对象
- `widget`: 小组件对象本身

#### scripts 函数
初始化脚本，在小组件创建后执行：

```javascript
"scripts": "console.log('初始化'); setupEventListeners();"
```

在脚本中可以使用：
- `contentEl`: 小组件的 DOM 元素
- `config`: 用户配置对象
- `widget`: 小组件对象本身

#### cleanup 函数
清理函数，在小组件销毁前执行：

```javascript
"cleanup": "clearInterval(interval);"
```

## 开发指南

### 1. 设计原则

- **响应式**: 小组件应能在不同尺寸下正常显示
- **性能**: 避免频繁的网络请求和重绘
- **用户友好**: 提供清晰的视觉反馈和交互
- **可配置**: 为常用设置提供配置选项

### 2. HTML 结构

使用语义化的 HTML 结构：

```html
<div class="my-widget">
  <div class="widget-header">
    <h3>标题</h3>
  </div>
  <div class="widget-content">
    <!-- 主要内容 -->
  </div>
</div>
```

### 3. CSS 样式

- 使用 CSS 变量支持主题切换
- 确保在不同尺寸下都能正常显示
- 使用 `height: 100%` 填充整个小组件区域

```css
.my-widget {
  height: 100%;
  padding: 16px;
  background: linear-gradient(135deg, #color1, #color2);
}
```

### 4. JavaScript 交互

#### 事件监听

```javascript
function setupInteractions() {
  const button = contentEl.querySelector('.my-button');
  button.addEventListener('click', () => {
    // 处理点击事件
  });
}
```

#### 数据更新

```javascript
async function updateData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    // 更新 DOM
    contentEl.querySelector('.data-display').textContent = data.value;
  } catch (error) {
    console.error('数据更新失败:', error);
  }
}
```

#### 定时更新

```javascript
function startUpdates() {
  updateData(); // 立即执行一次
  const interval = setInterval(updateData, 60000); // 每分钟更新
  contentEl._updateInterval = interval; // 保存引用用于清理
}
```

### 5. 权限处理

在使用需要权限的功能前，检查权限：

```javascript
if (widget.permissions && widget.permissions.includes('network')) {
  // 执行网络请求
}
```

## 示例小组件

### 简单时钟小组件

```json
{
  "type": "simple_clock",
  "author": "开发者",
  "name": "简单时钟",
  "size": "1x1",
  "desc": "显示当前时间",
  "category": "information",
  "version": "1.0.0",
  "html": "<div class=\"clock-widget\"><div class=\"time\">00:00:00</div></div>",
  "styles": ".clock-widget { height: 100%; display: flex; align-items: center; justify-content: center; background: #2c3e50; } .time { font-size: 18px; color: white; font-family: monospace; }",
  "scripts": "function updateTime() { const el = contentEl.querySelector('.time'); el.textContent = new Date().toLocaleTimeString(); } updateTime(); const interval = setInterval(updateTime, 1000); contentEl._clockInterval = interval;",
  "cleanup": "if (contentEl._clockInterval) { clearInterval(contentEl._clockInterval); }"
}
```

### 可配置天气小组件

```json
{
  "type": "weather",
  "author": "开发者",
  "name": "天气",
  "size": "1x2",
  "desc": "显示天气信息",
  "category": "information",
  "version": "1.0.0",
  "html": "<div class=\"weather-widget\"><div class=\"temp\">--°C</div><div class=\"desc\">加载中...</div></div>",
  "styles": ".weather-widget { height: 100%; padding: 16px; background: linear-gradient(135deg, #667eea, #764ba2); display: flex; flex-direction: column; justify-content: center; } .temp { font-size: 24px; color: white; text-align: center; } .desc { font-size: 14px; color: rgba(255,255,255,0.8); text-align: center; margin-top: 8px; }",
  "config": [
    {
      "key": "city",
      "label": "城市",
      "type": "text",
      "default": "北京"
    },
    {
      "key": "unit",
      "label": "温度单位",
      "type": "select",
      "options": ["celsius", "fahrenheit"],
      "default": "celsius"
    }
  ],
  "permissions": ["network"],
  "scripts": "async function fetchWeather() { try { const city = config.city || '北京'; const unit = config.unit === 'fahrenheit' ? 'F' : 'C'; const response = await fetch(`https://wttr.in/${city}?format=%t+%C`); const text = await response.text(); const [temp, desc] = text.split(' '); contentEl.querySelector('.temp').textContent = temp; contentEl.querySelector('.desc').textContent = desc; } catch (err) { contentEl.querySelector('.temp').textContent = '--°'; contentEl.querySelector('.desc').textContent = '获取失败'; } } fetchWeather(); const interval = setInterval(fetchWeather, 1800000); contentEl._weatherInterval = interval;",
  "cleanup": "if (contentEl._weatherInterval) { clearInterval(contentEl._weatherInterval); }"
}
```

## 调试技巧

### 控制台调试

```javascript
console.log('配置:', config);
console.log('小组件:', widget);
console.log('DOM 元素:', contentEl);
```

### 样式调试

- 验证 CSS 是否正确应用
- 检查响应式布局

### 功能测试

- 测试所有配置选项
- 验证权限检查
- 测试清理函数是否正确执行

## 发布指南

1. 测试小组件在所有支持的尺寸下都能正常工作
2. 验证配置选项的保存和恢复
3. 检查权限请求是否合理
4. 编写清晰的描述和使用说明
5. 为小组件选择合适的分类

## 最佳实践

- **性能优化**: 避免不必要的 DOM 操作和网络请求
- **错误处理**: 为所有异步操作添加错误处理
- **内存管理**: 正确清理定时器和事件监听器
- **可访问性**: 为交互元素添加适当的 ARIA 属性
- **国际化**: 为文本内容使用 i18n 支持

## 常见问题

### Q: 小组件不显示？
A: 检查 HTML 和 CSS 是否正确，验证 size 字段是否有效。

### Q: 配置不生效？
A: 确保 config 数组格式正确，重启应用后配置才会生效。

### Q: 网络请求失败？
A: 检查 permissions 字段是否包含 'network'，验证 API 地址是否正确。

### Q: 内存泄漏？
A: 确保在 cleanup 函数中清理所有定时器和事件监听器。
